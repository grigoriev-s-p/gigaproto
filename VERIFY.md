# Проверка интеграции

В этой версии проекта объединены:
- система рекомендаций после генерации UI
- система правок уже сгенерированного прототипа
- мост между ними: если пользователь отвечает на рекомендации фразой вроде «да, сделай так», рекомендации применяются как реальные правки через edit-механизм
- если пользователь пишет «нет не надо» или даёт свои правки, рекомендации не применяются, приоритет у текста пользователя

Что изменено:
- backend/fast_api.py
- backend/recommendation_edit_bridge.py
- backend/ui_edit_agent.py
- frontend/src/app/App.tsx
- frontend/src/app/types.ts

Что дополнительно улучшено после повторного сравнения двух архивов:
- сохранён более надёжный merge-движок правок из текущего проекта
- в его prompt добавлены сильные эвристики из второго архива:
  - чистка мусорных/технических подписей
  - требование русского языка для видимых текстов
  - более явные правила для кнопок, filters/actions и цветовых правок

Почему не заменён весь edit-механизм целиком на вариант из второго архива:
- во втором архиве редактор меняет только ui_preview
- он не синхронизирует правки с ui_schema
- он не делает безопасный merge поверх текущей структуры так глубоко, как текущий движок
- из-за этого он хуже подходит для связки с рекомендациями и для сохранения многостраничного прототипа

Что проверено:
- frontend: npm install + npm run build
- backend: локальный тест /edit в трёх сценариях
  1. обычная пользовательская правка
  2. подтверждение рекомендаций «да, сделай так»
  3. отказ от рекомендаций «нет не надо»
